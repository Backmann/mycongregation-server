## Conventions & gotchas — MUST READ before delivering code

- **Paste corruption in MINGW64.** Multi-line heredoc paste scrambles
  (bracketed-paste). NEVER deliver node scripts or commit messages via pasted
  heredoc. Deliver scripts AND commit-msg.txt as FILES inside a tarball; Lionel
  extracts and runs `node script.cjs` / `git commit -F commit-msg.txt`. Even
  `node -e "…"` with parentheses gets mangled — use a script file instead.
- **`git add` with explicit paths, NEVER `git add -A` for app commits.** Or
  `rm commit-msg.txt` BEFORE staging. A stray `commit-msg.txt` leaked into
  `4a0c1b1` because the apply steps ran `git add -A` before `rm`.
- **Node patch scripts:** CRLF-tolerant (try `\n` anchor, fall back to `\r\n`)
  and idempotent (check for a marker, skip if present). Files have LF working-
  tree endings (git warns "LF will be replaced by CRLF" — harmless).
- **Full files Claude has complete content for → deliver whole (overwrite).**
  Files Claude only has excerpts of (e.g. lib/api.ts ~900 lines) → deliver a
  node patch script with unique anchors.
- **i18n drift gate (app CI):** every key must exist in ru/en/de. Add keys
  programmatically via a node script (JSON.parse → set → `JSON.stringify(obj,
  null, 2) + '\n'`). Locale files are 2-space indent, **raw Cyrillic** (not
  escaped), so stringify matches and the diff stays minimal. Plural variants
  `_zero/_two/_few/_many` are exempt; `_one/_other` are required.
- **expo-server-sdk is ESM; ts-jest cannot parse it.** Any new server spec that
  transitively imports PublishersService (→ push-notifications.service →
  expo-server-sdk) MUST add, at the very top,
  `jest.mock('expo-server-sdk', () => ({ Expo: MockExpo }))` (jest hoists it).
  Pattern: `src/push-notifications/push-notifications.service.spec.ts`.
- **CI gates:** server = `npm test` (jest) only — lint NOT in server CI yet
  (P1 #17). App = `npm run lint` (expo lint) + `npm run i18n:check`. Server has
  husky+lint-staged (prettier --write on staged src/**/*.ts). App has no husky.
- **Server lint:** `npm run lint` auto-fixes (`--fix`); use `npx eslint --no-fix`
  for diagnostics. Baseline = 48 problems.
- **Workflow:** Claude generates tarball → Lionel SCPs via ~/Downloads →
  extract in `~/congmap/app` or `~/congmap/server` → run scripts → verify
  (tsc/lint/i18n or jest) → commit `-F commit-msg.txt` → push → CI deploys ~90s.
