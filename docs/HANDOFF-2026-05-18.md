## Conventions and gotchas learned in this session

### Build / deploy gotchas
- **Metro/Babel cache + env vars**: when env vars change, `rm -rf .expo node_modules/.cache dist` is mandatory or stale bundles ship. Today's CI workflow does this automatically.
- **`.env.production` vs `.env`**: Expo loads both when `NODE_ENV=production` (which `expo export -p web` sets by default). `.env.production` overrides `.env`. **Inline `VAR=value command` may not work on MINGW64 bash** — use `export` or `.env.production`.
- **Bind mount inode**: never `mv` the mount-point directory (`/var/www/mycongregation/`). Docker bind mounts attach to inode at container start, not path. Replace contents in-place: `rm -rf ./* && tar xzf`. Today's CI does this correctly.
- **Heredoc inside `ssh "..."` on MINGW64**: tends to mangle. Pipe SQL via `cat <<EOF | ssh ...` or use one-liners with `psql -c "..."`.
- **`bash !` history expansion**: bash treats `!` in double-quoted strings as history expansion. Passwords/values with `!` need single quotes or `set +H` disable.

### Workflow
- Both repos auto-deploy on push to `main`.
- Server CI gate: `npm test` (254 tests). App CI gate: `npm run lint`.
- Commit messages: Conventional Commits (`feat:`, `fix:`, `chore:`, `docs:`, `ci:`).

### Conventions
- **TypeORM `synchronize: false`** — all schema changes are manual SQL migrations under `src/migrations/`.
- **Multi-tenancy** — every entity has `congregationId`. Queries filter by `@TenantId()` decorator.
- **Soft-delete** — `deletedAt` column, excluded by default in queries.
- **Push channels** — Expo native AND Web Push are sent from the same `sendStatusChange` call; `excludeUserId` is the publisher's own `userId` (so they don't get notified about their own status change).

### App-side env (Expo)
- `__DEV__` global is `true` in dev, `false` in production builds (set by Metro).
- `process.env.EXPO_PUBLIC_*` vars are **inlined at build time** — they're not runtime. Changing them requires rebuild + cache clear.
- Dead-code elimination removes branches that reference inlined constants — that's why our production guard string ("must be set for production") doesn't appear in the shipped bundle: the `throw` branch is unreachable when `EXPO_PUBLIC_API_URL` is set.

### Sensitive values (already in scope, don't re-share verbatim)
- `KEK_BASE64` (encryption key on server `.env`, ready for Phase L)
- `VAPID_PUBLIC_KEY` / `VAPID_PRIVATE_KEY` (server `.env` + GitHub secret for public)
- DB: `POSTGRES_USER=mycongregation_user`, `POSTGRES_DB=mycongregation_db`, port 5433
